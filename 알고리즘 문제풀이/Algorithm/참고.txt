복잡도는 알고리즘 성능을 나타내는 척도

- 시간복잡도: 특정 크기의 입력에 대해서 알고리즘의 수행시간 분석
- 공간복잡도: 특정한 크기의 입력에 대해 알고리즘의 메모리 사용량 분석

빅오 표기법

- O(1): 상수 시간
- O(lgN): 로그 시간
- O(N): 선형 시간
- O(NlgN): 로그 선형시간
- O(N^2): 이차 시간
- O(N^3): 삼차 시간
- O(2^n): 지수 시간

연산 횟수가 5억을 넘어가는 경우
- C 언어 기준으로 1~3초 시간이 소요
- Python 경우 5~15초 소요

O(n^3) 알고리즘을 설계한 경우 N의 값이 5000을 넘긴다면 시간초과
시간 제한은 통상 1~5초 가량을 유의

문제풀이시 시간 제한을 주의해야 하며
시간제한이 1초인 문제를 만날 경우 일반적인 기준은 다음과 같다

- N의 범위가 500인 경우 O(n^3)인 알고리즘 작성시 문제를 풀 수 있음
- N의 범위가 2000인 경우 O(n^2)인 알고리즘 작성시 문제를 풀 수 있음
- N의 범위가 100000인 경우 O(nlgN)인 알고리즘 작성시 문제를 풀 수 있음
- N의 범위가 10000000인 경우 O(N)인 알고리즘을 작성시 문제를 풀 수 있음

완전탐색과 dp를 구분법 
일반적으로 문제가 큰문제랑 작은 문제로 나눠지며 작은 문제에서 구한 값이 큰문제에서도 똑같은 답으로 쓰인다면 dp로 접근가능

개발형 코딩테스트

-완성도 높은 하나의 프로그램을 개발
-모듈을 적절히 조합하는 능력을 요구

분야에 따라 상세 요구사항이 달라질수 있다.
